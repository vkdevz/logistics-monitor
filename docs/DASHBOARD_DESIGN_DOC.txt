REAL-TIME LOGISTICS BOTTLENECK MONITOR
Complete Dashboard Design Document (Tableau / Power BI)
================================================================================

PROJECT OVERVIEW:
This design doc provides exact specifications for building a production-grade 
logistics analytics dashboard. It includes 8 interconnected visualizations,
filter hierarchy, drill-down capabilities, and KPI tracking.

Expected refresh rate: Real-time (via live connector) or hourly batch updates

================================================================================
EXECUTIVE DASHBOARD (Dashboard 1: Overview)
================================================================================

PAGE LAYOUT: 3-column grid (1200px wide)
BACKGROUND: Dark gray (#1a1a1a) with white text
THEME: Corporate analytics (clean, data-forward)

--- KPI CARDS (Top Row - 4 Cards, Equal Width) ---

KPI 1: Total Shipments
  • Value: COUNT(Shipment_ID)
  • Format: 5,000 (large font, #4CAF50 green)
  • Sub-text: "Active in system" (gray, small)
  • Data source: shipments_live_data.csv

KPI 2: Average Delay (Hours)
  • Value: AVG(Delay_Hours)
  • Format: 12.4h (large font, #FF9800 orange if >12h, #4CAF50 if <12h)
  • Sub-text: "↑ +3.2% vs yesterday" (red trending up)
  • Data source: shipments_live_data.csv

KPI 3: Financial Loss (USD)
  • Value: SUM(Financial_Loss_USD)
  • Format: $31.4M (large font, #F44336 red)
  • Sub-text: "10% reduction target: $3.14M"
  • Data source: shipments_live_data.csv

KPI 4: On-Time Rate (%)
  • Value: COUNTIF(Status="On-Time") / COUNT(Shipment_ID) * 100
  • Format: 70.0% (large font, #2196F3 blue)
  • Sub-text: "Target: 85%" (gray)
  • Data source: shipments_live_data.csv

--- VISUALIZATION 1: Regional Delay Heatmap ---

Chart Type: HEAT MAP
Position: Left column, Row 2 (spans 1/3 width)
Dimensions:
  • Rows (Y-axis): Region (APAC, EMEA, AMER, LATAM)
  • Columns (X-axis): Status (On-Time, Delayed, Critical Delay)
  
Measures:
  • Color intensity: SUM(Financial_Loss_USD)
  • Cell values: AVG(Delay_Hours) (displayed as text overlay)

Color Scheme:
  • Light color (white/light gray): $0-5M
  • Medium color (orange): $5M-15M
  • Dark color (red): $15M+

Filters Applied:
  • Time range selector (default: last 7 days)
  • Status filter (checkboxes)

Expected output: 4 rows × 3 columns matrix showing loss heat by region/status

--- VISUALIZATION 2: Delay Trend (Line Chart) ---

Chart Type: LINE CHART
Position: Middle column, Row 2 (spans 1/3 width)
Dimensions:
  • X-axis: Ship_Date (DATE, binned to daily)
  • Y-axis (Left): AVG(Delay_Hours)
  • Y-axis (Right): SUM(Financial_Loss_USD)

Lines:
  • Blue line: Average delay (left Y-axis, thick)
  • Orange line: Total daily loss (right Y-axis, thin)

Formatting:
  • Grid lines: Subtle gray
  • Data point markers: Small circles
  • Tooltip: Show date, avg delay, total loss, shipment count

Filters:
  • Region filter (multi-select)
  • Date range picker

Expected output: 7-day trend showing delay patterns and financial impact

--- VISUALIZATION 3: Top 10 Problem Routes ---

Chart Type: HORIZONTAL BAR CHART
Position: Right column, Row 2 (spans 1/3 width)
Dimensions:
  • Y-axis (Rows): Route (sorted descending)
  • X-axis (Values): SUM(Financial_Loss_USD)

Bars:
  • Color by: AVG(Delay_Hours)
  • Color gradient: Green (8h) → Yellow (12h) → Red (16h+)
  • Bar labels: Show dollar amounts

Data:
  • Only show Top 10 routes by total loss
  • Exclude on-time shipments (Status = "Delayed" OR "Critical Delay")

Tooltip fields:
  • Route, Total Loss, Avg Delay, Shipment Count, Delay Rate %

Expected output: Top 10 routes ranked by impact, color-coded by severity

--- VISUALIZATION 4: Carrier Performance Matrix ---

Chart Type: SCATTER PLOT
Position: Top-left (50% width)
Dimensions:

  • Y-axis: SUM(Financial_Loss_USD) - labeled "Total Financial Loss ($)"

Dimensions:

================================================================================
FILTER HIERARCHY & INTERACTIONS
================================================================================

PRIMARY FILTERS (Global, apply across all dashboards):

1. Date Range Selector
   Default: Last 7 days
   Options: Last 24h, Last 7d, Last 30d, Custom date range
   Location: Top-left of every dashboard
   Type: Date picker with calendar

2. Region Filter
   Options: APAC, EMEA, AMER, LATAM (multi-select checkboxes)
   Default: All regions selected
   Behavior: Filters all visualizations except KPI summary cards
   Location: Top-center
   Type: Checkbox buttons

3. Carrier Filter
   Options: FedEx, DHL, UPS, Maersk, Hapag-Lloyd, CMA CGM, Local Courier
   Default: All carriers selected
   Behavior: Filters route table and scatter plot
   Location: Top-center
   Type: Multi-select dropdown

4. Status Filter
   Options: On-Time, Delayed, Critical Delay, In Transit, Pending
   Default: All statuses (but can highlight one)
   Behavior: Color-codes and separates visualizations
   Location: Top-right
   Type: Segmented buttons (On/Off per status)

SECONDARY INTERACTIONS (Drill-down):

• Click Route in bar chart → Route detail table updates to show only that route
• Click Region in heat map → Carrier scatter plot filters to that region
• Click Carrier bubble in scatter → Route table filters to that carrier's routes
• Click Row in Route table → Pop-up detail view shows all shipments on that route

Expected flow: Users can filter globally, then drill into specific dimensions

DATA SOURCE CONNECTIONS (Tableau/Power BI)
================================================================================

FLAT FILES (CSV):
├── shipments_live_data.csv (Primary source, 5,000 rows)
│   Fields: Shipment_ID, Region, Route, Carrier, Status, Shipment_Value_USD,
│            Delay_Hours, Financial_Loss_USD, Ship_Date, Expected_Delivery,
│            Current_Location, Timestamp
│
├── regional_analytics.csv (Pre-aggregated for performance)
│   Fields: Region, Avg_Delay_Hours, Max_Delay_Hours, Total_Delay_Hours,
│            Total_Financial_Loss_USD, Avg_Financial_Loss_USD,
│            Total_Shipments, Delayed_Count, Delay_Rate_%
│
└── carrier_analytics.csv (Pre-aggregated for performance)
    Fields: Carrier, Avg_Delay_Hours, Total_Financial_Loss_USD,
            Avg_Financial_Loss_USD, Total_Shipments

CALCULATED FIELDS (Create in Tableau/Power BI):

Calc Field 1: On_Time_Rate
  Formula: COUNTIF([Status] = "On-Time") / TOTAL(COUNT([Shipment_ID])) * 100
  Type: Percent
  Use: KPI cards, gauges

Calc Field 2: Delay_Category
  Formula: IF [Delay_Hours] = 0 THEN "On-Time"
           ELSEIF [Delay_Hours] < 24 THEN "Delayed"
           ELSE "Critical Delay"
  Type: String
  Use: Grouping, color-coding

Calc Field 3: Loss_Per_Hour
  Formula: [Financial_Loss_USD] / ([Delay_Hours] + 0.1)  // avoid div by zero
  Type: Currency
  Use: Scatter plot, trend analysis

Calc Field 4: Region_Rank
  Formula: RANK([Avg_Delay_Hours]) OVER [Region]
  Type: Integer
  Use: Identify top problem regions

REFRESH & PERFORMANCE OPTIMIZATION
================================================================================

DATA REFRESH STRATEGY:

Option 1 (Live API - Recommended):
  • Connect Tableau/Power BI directly to API or database
  • Refresh every 15 minutes (or real-time for critical metrics)
  • Use incremental loads to avoid re-fetching all 5,000 rows

Option 2 (Batch Updates):
  • Hourly upload of shipments_live_data.csv via automated job
  • Pre-aggregate to regional_analytics.csv and carrier_analytics.csv
  • Reduces dashboard load time by 60%

PERFORMANCE BEST PRACTICES:

1. Aggregate Where Possible
   • Use regional_analytics.csv for summary metrics
   • Drill-down to shipments_live_data.csv on demand only

2. Limit Row Limits on Tables
   • Show max 100 rows in tables (paginate)
   • Use "Show Top 10/20" on bar charts

3. Index Key Fields
   • Ensure Region, Carrier, Route are indexed
   • Pre-sort by Date and Financial_Loss for fast rendering

4. Caching
   • Cache hourly aggregations in memory
   • Use Tableau extract for heavy calculations

Expected dashboard load time: <2 seconds
