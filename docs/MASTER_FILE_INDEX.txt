────────────────────────────────────────────────────────────────────────────────
Size: 560 KB | Rows: 5,000 | Columns: 12
Format: Comma-separated values (standard CSV)

COLUMN DEFINITIONS:
  1. Shipment_ID ................. Unique identifier (SHP000001-SHP005000)
  2. Region ...................... APAC, EMEA, AMER, or LATAM
  3. Route ....................... Source-Destination (e.g., SG-JP)
  4. Carrier ..................... FedEx, DHL, UPS, etc. (7 options)
  5. Status ...................... On-Time, Delayed, Critical Delay, etc.
  6. Shipment_Value_USD .......... Dollar value of shipment (5K-50K range)
  7. Delay_Hours ................. Hours of delay (0 to 142 range)
  8. Financial_Loss_USD .......... Calculated loss per shipment
  9. Ship_Date ................... Date/time shipment was sent
  10. Expected_Delivery .......... Expected arrival date
  11. Current_Location ........... Port/hub identifier
  12. Timestamp .................. Record generation time

DATA CHARACTERISTICS:
  • 70% on-time, 20% delayed, 10% critical delay (realistic distribution)
  • Delay variance: ~24 hours standard deviation
  • Loss range: $0 (on-time) to $87K (critical delays)
  • 7-day date range (May 15-22, 2026)
  • 4 regions with realistic volume ratios
  • 7 carriers with measurable performance variance

USE CASES:
  1. Primary data source for Tableau/Power BI dashboards
  2. Load directly without transformation
  3. Backup/test data for validation
  4. Reference for production data structure design

HOW TO USE:
  Tableau: Data Source > Add Connection > Text File > Select CSV
  Power BI: Get Data > CSV > Select file
  Excel: File > Open > Select CSV
  
VALIDATION:
  • No null/missing values (production-ready)
  • All dates within 7-day range
  • Delay hours >= 0
  • Financial loss >= 0
  • Region in (APAC, EMEA, AMER, LATAM)
